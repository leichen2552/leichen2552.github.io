SSD 固件的角度来看，一般可以把整个数据路径分成三部分：</span></p><ul class="list-paddingleft-1"><li><p><strong><span leaf="">前端</span></strong><span leaf="">：和 Host 打交道</span></p></li><li><p><strong><span leaf="">通常在点击保存的那一刻，数据先进入的是&nbsp;</span><strong><span leaf="">系统内存（DDR）</span></strong><span leaf="">，并没有立刻落到 SSD 里。</span><span leaf=""><br></span><span leaf="">当关闭 写大概是这样走的：</span></p><h3><span leaf=""><span textstyle="" style="font-size: 18px;font-weight: bold;">对前端来说：</span></span></h3><p><span leaf="">前端负责接收 Host 发下来的写命令和写数据。</span><span solid;border-width: 1px 0 0;border-color: rgba(0,0,0,0.1);-webkit-transform-origin: 0 0;-webkit-transform: scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span textstyle="" style="font-size: 20px;font-weight: bold;">为什么写下去之后的数据不是一个“点”，而是一个分布？</span></span></h1><p><span leaf="">这个问题，我自己也查了不少资料，后来才慢慢想明白。</span></p><p><span leaf="">先从最简单的情况开始。</span></p><p><span leaf="">我们知道，program 操作是按照&nbsp;</span><strong><span leaf="">page 顺序</span></strong><span leaf="">&nbsp;从小到大依次进行的。</span><span leaf="">目前常见的 page 大小是&nbsp;</span><strong><span leaf="">16KB</span></strong><span leaf="">。</span></p><p><span leaf="">一个 page 是由很多个浮栅晶体管组成的，也就是我们常说的&nbsp;</span><strong><span leaf="">cell</span></strong><span class="list-paddingleft-1"><li><p><span leaf="">一个 cell 只存储 1 bit</span></p></li><li><p><span leaf="">所以一个 cell 只可能表示 0 或 1</span></p></li></ul><p><span leaf="">如果一个 page 是 16KB，那么它一共有：</span></p><pre><code><span leaf="">16 × 1024 × 8 = 131072 bit</span><span leaf=""><br></span></code></pre><p><span leaf="">也就是说，这一个 page 里，大概就有&nbsp;</span><strong><span leaf="">131072 个 cell</span></strong><span leaf="">。</span></p><hr style="border-style: solid;border-width: 1px 0 0;border-color: rgba(0,0,0,0.1);-webkit-transform-origin: 0 0;-webkit-transform: scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span textstyle="" style="font-size: 20px;font-weight: bold;">我们直觉上理解的“写入”，其实是不对的</span></span></h1><p><span leaf="">之前文章里提到过，program 的本质就是加高电压，把电子打进浮栅层里。</span></p><p><span leaf="">如果站在直觉上理解，很多人会觉得：</span></p><blockquote><p><span leaf="">对于 SLC 来说，只要这个 cell 里电子数量超过某个门限，比如“超过 50%”，那它就可以表示成 1。</span></p></blockquote><p><span leaf="">所以看起来好像很简单：</span></p><ul class="list-paddingleft-1"><li><p><span leaf="">加高电压</span></p></li><li><p><span leaf="">把电子打进去</span></p></li><li><p><span leaf="">超过阈值就行了</span></p></li></ul><p><span leaf="">但 NAND 的真实操作和我们这种直觉其实是反着来的。</span></p><p><strong><span leaf="">它不是在“写 bit”，不是在写入 0 或 1。</span></strong><span leaf=""><br></span><span leaf="">它真正做的事情是：</span></p><blockquote><p><strong><span leaf="">向浮栅层中注入电子，改变这个 cell 的阈值电压。</span></strong></p></blockquote><p><span leaf="">这是一个</span><strong><span leaf="">模拟过程</span></strong><span leaf="">，不是数字过程。</span></p><p><span leaf="">这一点非常关键。</span></p><hr style="border-style: solid;border-width: 1px 0 0;border-color: rgba(0,0,0,0.1);-webkit-transform-origin: 0 0;-webkit-transform: scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span textstyle="" style="font-size: 20px;font-weight: bold;">Program 不是一次打进去，而是 Program + Verify</span></span></h1><p><span leaf="">NAND 的 program 并不是：</span></p><blockquote><p><span leaf="">一次加高压，直接把 cell 写成目标状态。</span></p></blockquote><p><span leaf="">而是采用一种&nbsp;</span><strong><span leaf="">Program + Verify</span></strong><span leaf="">&nbsp;的方式。</span></p><p><span leaf="">原因很简单：</span></p><blockquote><p><span leaf="">它并不知道每次加电压到底能打进去多少电子。</span></p></blockquote><p><span leaf="">如果还没到目标阈值，就再打一小次</span></p></li><li><p><span leaf="">再 verify</span></p></li><li><p><span leaf="">如此反复，直到超过目标阈值为止</span></p></li></ul><p><span leaf="">你可以把它理解成往一个水桶里加水。</span></p><p><span leaf="">假设水桶高 20cm，我们的目标不是加满，而是：</span></p><blockquote><p><span leaf="">只要水位超过 10cm 就算成功</span></p></blockquote><p><span leaf="">那真实过程就不会是“一次灌到 10cm”。</span><span leaf=""><br></span><span leaf="">而更像是：</span></p><ul class="list-paddingleft-1"><li><p><span leaf="">先倒一点</span></p></li><li><p><span leaf="">用尺子量一下</span></p></li><li><p><span leaf="">不够，再倒一点</span></p></li><li><p><span leaf="">逐步逼近目标阈值</span></strong><span leaf="">&nbsp;的过程</span></p></blockquote><p><span leaf="">而不是一个“一步到位”的过程。</span></p><hr style="border-style: solid;border-width: 1px 0 0;border-color: rgba(0,0,0,0.1);-webkit-transform-origin: 0 0;-webkit-transform: scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span textstyle="" style="font-size: 20px;font-weight: bold;">那为什么最后的电压会形成一个分布？</span></span></h1><p><span leaf="">理解这一点，先要搞清楚一个反例。</span></p><p><span leaf="">假设一个 page 里的那&nbsp;</span><strong><span leaf="">131072 个 cell</span></strong><span leaf="">&nbsp;是完全一模一样的。</span></p><p><span leaf="">这种“完全一样”包括：</span></p><ul class="list-paddingleft-1"><li><p><span leaf="">每个 cell 的初始状态完全一样</span></p></li><li><p><span leaf="">每个 cell 的氧化层厚度完全一样</span></p></li><li><p><span leaf="">每个 cell 的电容完全一样</span></p></li><li><p><span leaf="">每个 cell 对每次电压脉冲的响应完全一样</span></p></li><li><p><span leaf="">Verify 判断完全没有误差</span></p></li><li><p><span leaf="">周围环境完全一样</span></p></li><li><p><span leaf="">温度完全一样</span></p></li><li><p><span leaf="">噪声完全一样</span></p></li></ul><p><span leaf="">如果这些条件全部成立，那么最后 program 结束时，这 131072 个 cell 的阈值电压就会完全一样。</span></p><p><span leaf="">那这时候，如果你去统计：</span></p><ul class="list-paddingleft-1"><li><p><span leaf="">横轴：Vth</span></p></li><li><p><span leaf="">纵轴：cell 数量</span></p></li></ul><p><span leaf="">最后看到的就不是一个山包，而应该是一个&nbsp;</span><strong><span leaf="">点</span></strong><span leaf="">。</span></p><p><span leaf="">因为所有 cell 都落在同一个电压上。</span></p><hr style="border-style: solid;border-width: 1px 0 0;border-color: rgba(0,0,0,0.1);-webkit-transform-origin: 0 0;-webkit-transform: scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span textstyle="" style="font-size: 20px;font-weight: bold;">但现实中的 NAND，不可能做到每个 cell 都一模一样</span></span></h1><p><span leaf="">真实世界里，NAND 的制造工艺不可能把所有 cell 做到完全一致。</span></p><p><span leaf="">这些差异虽然非常小，但在 NAND 这种极其敏感的模拟存储器里，会被放大到阈值电压上。</span></p><p><span leaf="">我把这些原因分成三类。</span></p><hr style="border-style: solid;border-width: 1px 0 0;border-color: rgba(0,0,0,0.1);-webkit-transform-origin: 0 0;-webkit-transform: scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span textstyle="" style="font-size: 18px;font-weight: bold;">1. 工艺差异</span></span></h1><p><span leaf="">比如说：</span></p><ul class="list-paddingleft-1"><li><p><span leaf="">隧穿氧化层厚一点、薄一点</span></p></li><li><p><span leaf="">沟道长度有轻微差别</span></p></li><li><p><span leaf="">掺杂浓度有轻微差别</span></p></li><li><p><span leaf="">浮栅尺寸有轻微差别</span></p></li><li><p><span leaf="">电容耦合比有轻微差别</span></p></li></ul><p><span leaf="">这些差异单独看都很小，甚至小到工艺上觉得“很正常”。</span></p><p><span leaf="">但 NAND program 本来就是一个非常细腻的模拟过程，所以这些微小差异最后都会反映到 (Vth) 上。</span></p><p><span leaf="">你可以把它理解成：</span></p><p><span leaf="">同样打一拳，</span></p><ul class="list-paddingleft-1"><li><p><span leaf="">有的 cell “涨 0.15V”</span></p></li><li><p><span leaf="">有的 cell “涨 0.18V”</span></p></li><li><p><span leaf="">有的 cell “涨 0.12V”</span></p></li></ul><p><span leaf="">同样的 program脉冲，最后每个 cell 的响应其实是不一样的。</span></p><hr style="border-style: solid;border-width: 1px 0 0;border-color: rgba(0,0,0,0.1);-webkit-transform-origin: 0 0;-webkit-transform: scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span textstyle="" style="font-size: 18px;font-weight: bold;">2. 电子注入本身就是随机的</span></span></h1><p><span leaf="">电子不是整整齐齐排队进去的。</span></p><p><span leaf="">Program 依赖的是高电场下的电子隧穿，而隧穿这件事本身就带有统计随机性。</span></p><p><span leaf="">换句话说：</span></p><blockquote><p><span solid;border-width: 1px 0 0;border-color: rgba(0,0,0,0.1);-webkit-transform-origin: 0 0;-webkit-transform: scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span (Vth)：</span></p><ul class="list-paddingleft-1"><li><p><span leaf="">更靠左一点</span></p></li><li><p><span leaf="">有的更靠右一点</span></p></li></ul><p><span leaf="">所以即使后面受到相同的 program 厘米</span></p></li></ul><p><span leaf="">最后大家冲线的位置，当然不可能整整齐齐在同一个点上。</span></p><hr style="border-style: solid;border-width: 1px 0 0;border-color: rgba(0,0,0,0.1);-webkit-transform-origin: 0 0;-webkit-transform: scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span textstyle="" style="font-size: 20px;font-weight: bold;">为什么有 Verify scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span textstyle="" style="font-size: 18px;font-weight: bold;">用“停车”来理解 Verify</span></span></h1><p><span leaf="">假设现在有 10000 辆车要停进同一个停车位区域。</span></p><p><span leaf="">规则是：</span></p><ul class="list-paddingleft-1"><li><p><span leaf="">只要车头进到白线以内，就算合格</span></p></li><li><p><span 的本质。</span></p><hr style="border-style: solid;border-width: 1px 0 0;border-color: rgba(0,0,0,0.1);-webkit-transform-origin: 0 0;-webkit-transform: scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span textstyle="" style="font-size: 20px;font-weight: bold;">Program + Verify 真正做的，不是“写成同一个值”</span></span></h1><p><span leaf="">Program + Verify 真正做的事情并不是把所有 cell 放在一起统计出来的分布。</span></strong></p></blockquote><p><span leaf="">对单个 cell 来说，在某一个时刻，它只有一个实际的阈值电压值。</span><span leaf=""><br></span><span leaf="">比如：</span></p><ul class="list-paddingleft-1"><li><p><span leaf="">2.31V</span></p></li><li><p><span leaf="">3.02V</span></p></li><li><p><span leaf="">1.87V</span></p></li></ul><p><span leaf="">它本身不是一座山。</span></p><p><span leaf="">真正的“山包图”是这样来的：</span></p><ul class="list-paddingleft-1"><li><p><span leaf="">把一个 page 里的很多个 cell 都 program 到同一个状态</span></p></li><li><p><span leaf="">然后统计这些 cell 的 Vth</span></p></li><li><p><span leaf="">看看有多少个落在 2.8V 附近</span></p></li><li><p><span leaf="">有多少个落在 2.9V 附近</span></p></li><li><p><span leaf="">有多少个落在 3.0V 附近</span></p></li></ul><p><span leaf="">最后才画出一个电压分布图。</span></p><p><span leaf="">所以图里的纵轴一般写的是：</span></p><pre><code><span leaf=""># of cells</span><span leaf=""><br></span></code></pre><p><span leaf="">也就是：</span></p><p><strong><span leaf="">在这个电压附近，有多少个 cell。</span></strong></p><hr style="border-style: solid;border-width: 1px 0 0;border-color: rgba(0,0,0,0.1);-webkit-transform-origin: 0 0;-webkit-transform: scale(1, 0.5);transform-origin: 0 0;transform: scale(1, 0.5);"><h1><span leaf=""><span textstyle="" style="font-size: 20px;font-weight: bold;">为什么这个问题这么重要？</span></span></h1><p><span leaf="">因为 NAND 的读取，本质上不是“直接读出 0 和 1”。</span></p><p><span leaf="">而是：</span></p><blockquote><p><strong><span leaf="">拿一个参考电压去判断，这个 cell 当前的 Vth 落在哪个区间。</span></strong></p></blockquote><p><span leaf="">如果 program 后每个状态都是一个点，读取会非常简单。</span></p><p><span leaf="">但真实情况不是点，而是一团分布。</span><span leaf=""><br></span><span leaf="">而且随着：</span></p><ul class="list-paddingleft-1"><li><p><span leaf="">P/E 次数增加</span></p></li><li><p><span leaf="">Retention 变差</span></p></li><li><p><span leaf="">Program Disturb</span></p></li><li><p><span leaf="">Read Disturb</span></p></li></ul><p><span leaf="">这些分布还会继续漂移、变宽、靠近，甚至重叠。</span></p><p><span leaf="">这时候 read、read retry、ECC 才变得有意义。</span></p><p><span leaf="">所以如果 program 之后的分布这件事没想明白，后面的 NAND Read 和 Read Retry 其实很难真正看懂。</span></p><hr leaf=""><span textstyle="" style="font-size: 20px;font-weight: bold;">总结</span></span></h1><p><span leaf="">NAND program 完成之后，电压之所以会形成一个类似正态分布的图，不是因为单个 cell 自己会“长成一座山”，而是因为：</span></p><ol class="list-paddingleft-1"><li><p><span leaf="">一个 page 里有大量 cell</span></p></li><li><p><span leaf="">每个 cell 的工艺、响应、初始状态都不完全一样</span></p></li><li><p><span
